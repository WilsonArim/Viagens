# /plan — Do Plano a Arquitetura

> Workflow que transforma requisitos num plano tecnico detalhado.

## Skills Encadeadas

1. **concise-planning** — Estruturar o plano de trabalho
2. **senior-architect** — Definir arquitetura do sistema
3. **architecture-patterns** — Selecionar patterns adequados
4. **database-design** — Desenhar schema de dados

## Sequencia

### Passo 1: Planear (concise-planning)
- Definir objetivo claro
- Listar passos necessarios
- Identificar riscos e dependencias
- Definir criterios de sucesso

### Passo 2: Arquitetar (senior-architect)
- Definir camadas do sistema
- Mapear componentes e responsabilidades
- Definir interfaces entre componentes
- Considerar escalabilidade e manutencao

### Passo 3: Patterns (architecture-patterns)
- Selecionar pattern arquitetural (Clean, DDD, Hexagonal, ou simples)
- Definir estrutura de pastas
- Mapear dependencias entre modulos

### Passo 4: Dados (database-design)
- Desenhar schema da base de dados
- Definir relacoes e indices
- Planear migracoes

## Output Esperado

```
1. Plano de trabalho (Goal, Steps, Risks, Success Criteria)
2. Diagrama de arquitetura (texto)
3. Estrutura de pastas do projeto
4. Schema da base de dados
5. Lista de tarefas ordenada por prioridade
```

## Quando Usar

- "Planeia a arquitetura para..."
- "Como devo estruturar..."
- "Preciso de um plano tecnico para..."
- "Desenha o sistema para..."
