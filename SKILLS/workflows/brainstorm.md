# /brainstorm — Da Ideia ao Plano

> Workflow que transforma uma ideia vaga num plano concreto e acionavel.

## Skills Encadeadas

1. **brainstorming** — Explorar e refinar a ideia
2. **product-manager-toolkit** — Estruturar como PRD com priorizacao
3. **architecture-decision-records** — Documentar decisoes tecnicas chave

## Sequencia

### Passo 1: Divergir (brainstorming)
- Ouvir a ideia do utilizador
- Fazer perguntas clarificadoras
- Explorar variantes e possibilidades
- Identificar o problema real a resolver

### Passo 2: Convergir (brainstorming)
- Definir o MVP (must-have vs nice-to-have)
- Sugerir stack tecnologica
- Estimar complexidade (S/M/L)

### Passo 3: Estruturar (product-manager-toolkit)
- Criar PRD simplificado
- Definir user stories principais
- Aplicar priorizacao RICE

### Passo 4: Decidir (architecture-decision-records)
- Documentar decisoes tecnicas (stack, patterns, integracao)
- Registar tradeoffs e alternativas consideradas

## Output Esperado

```
1. Resumo da ideia (1 paragrafo)
2. PRD simplificado com features priorizadas
3. User stories principais (3-5)
4. Stack tecnologica recomendada com justificacao
5. ADRs para decisoes chave
6. Proximos passos concretos
```

## Quando Usar

- "Tenho uma ideia para..."
- "Quero construir..."
- "Preciso de um MVP para..."
- "O que achas de fazer X?"
